Perfetto! Ecco un manuale completo dei passaggi che abbiamo seguito finora per avere Duplicati funzionante su CasaOS, basato sui comandi che hai eseguito e sulla configurazione del container. Ho organizzato tutto in modo chiaro e sequenziale.

Manuale: Configurazione di Duplicati su CasaOS per backup del Raspberry

1. Preparazione del sistema

  1. Sistema operativo: Debian 12 (Bookworm) su Raspberry Pi aarch64.

  2. CasaOS installato e in esecuzione:

    bash
    systemctl status casaos

  3. Assicurati di avere spazio sufficiente per backup e configurazioni su /DATA e sulla chiavetta USB.

2. Montaggio della chiavetta USB per i backup

  1. Crea il punto di mount:

    bash
    sudo mkdir -p /mnt/usb

  2. Verifica il filesystem della chiavetta:

    bash
    lsblk -f

  3. Aggiorna /etc/fstab per mount automatico:

    text
    UUID=58C7-89DF /mnt/usb exfat defaults,uid=1000,gid=1000 0 0

  4. Monta tutte le voci di fstab:

    bash
    sudo mount -a

  5. Verifica:

    bash
    ls /mnt/usb

3. Creazione delle cartelle di lavoro

  1. Cartelle sul Raspberry per Duplicati:

    bash
    sudo mkdir -p /DATA/Duplicati/backups
sudo mkdir -p /DATA/Duplicati/source
sudo mkdir -p /DATA/AppData/duplicati/config

  2. Assicurati che siano accessibili dall’utente Duplicati:

    bash
    sudo chown -R 1000:1000 /DATA/Duplicati /DATA/AppData/duplicati

4. Configurazione del container Duplicati

  1. Variabili di ambiente:

    yaml
    - DUPLICATI__WEBSERVICE_INTERFACE=0.0.0.0
- DUPLICATI__WEBSERVICE_PASSWORD=@Duplicati240894
- PUID=0
- PGID=0
- SETTINGS_ENCRYPTION_KEY=@Duplicati240894
- TZ=Europe/London

  2. Mappatura dei volumi:

    yaml
    - /DATA/AppData/duplicati/config:/config
- /DATA/Duplicati/backups:/backups
- /DATA/Duplicati/source:/source
- /home/ulisse:/source/home
- /DATA:/source/DATA
- /etc:/source/etc
- /mnt/usb/RPiBackup:/backups

  3. Configurazione porta e rete:

    yaml
    ports:
  - target: 8200
    published: "8200"
    protocol: tcp
network_mode: host

5. Avvio e verifica

  1. Avvia il container:

    bash
    sudo docker start duplicati

  2. Controlla i log:

    bash
    sudo docker logs duplicati --tail 50

  3. Assicurati che Duplicati sia accessibile dal browser:

    text
    http://<IP_RPi>:8200

    Inserisci la password impostata in DUPLICATI__WEBSERVICE_PASSWORD.

6. Configurazione backup

  1. Seleziona sorgenti di backup:

    • /source/DATA (dati principali)

    • /source/home (home dell’utente)

    • /source/etc (configurazioni di sistema)

  2. Seleziona destinazione:

    • /backups → chiavetta USB o cartella di backup sul disco.

  3. Imposta opzioni di backup:

    • Backup intelligente.

    • Frequenza: ogni 12h (incrementale) o settimanale completo.

  4. Eventuali file in uso (es. database di container in esecuzione) possono generare warning:

    text
    Failed to process path: ... because it is being used by another process

    • Opzione: fermare temporaneamente i container per backup di database critici.

7. Test del backup

  1. Avvia manualmente un backup.

  2. Verifica log per errori o warning.

  3. Conferma che i backup siano visibili sulla chiavetta USB (/mnt/usb/RPiBackup).

8. Note importanti

  • Evita di far girare Duplicati come root; è meglio usare utente con PUID/PGID 1000 e permessi sulle cartelle di backup.

  • La cartella /source deve mappare tutte le directory critiche da salvare.

  • La chiavetta USB può essere utilizzata come storage di emergenza in caso di rottura microsd.

  • È consigliabile verificare periodicamente i backup tramite restore test.
