Perfetto! Creiamo un manuale passo passo per il ripristino completo del tuo Raspberry usando CasaOS e Duplicati, considerando che hai backup su chiavetta USB. Ti preparo la versione strutturata, chiara e con note di sicurezza.

Manuale di Ripristino del Raspberry

1. Preparazione dell’hardware

  1. Procurati il Raspberry e una microsd vuota o già formattata.

  2. Collega la chiavetta USB dove sono archiviati i backup.

2. Installazione del sistema operativo

  1. Scarica e installa Raspberry Pi OS (Debian Bookworm, 64-bit se hai aarch64) oppure l’OS su cui era CasaOS.

  2. Configura rete, SSH e account utente principale (ulisse) come era sul vecchio sistema.

3. Installazione di CasaOS

  1. Segui la procedura ufficiale CasaOS per installare CasaOS sul nuovo sistema.

  2. Verifica che il servizio CasaOS sia attivo:

    bash
    systemctl status casaos

  3. Se necessario, abilita CasaOS all’avvio:

    bash
    sudo systemctl enable casaos

4. Installazione di Duplicati

  1. Installa Duplicati via CasaOS App Store, come hai fatto in precedenza.

  2. Assicurati di mappare le cartelle corrette:

    • /source → la radice dei dati che vuoi ripristinare (/DATA, /home/ulisse, /etc, ecc.)

    • /backups → la chiavetta USB con i backup

  3. Inserisci le variabili di ambiente essenziali:

    yaml
    DUPLICATI__WEBSERVICE_INTERFACE=0.0.0.0
DUPLICATI__WEBSERVICE_PASSWORD=<la tua password>
PUID=1000
PGID=1000
SETTINGS_ENCRYPTION_KEY=<la tua chiave>
TZ=Europe/London

5. Montaggio della chiavetta USB

  1. Crea un punto di mount:

    bash
    sudo mkdir -p /mnt/usb

  2. Aggiorna /etc/fstab per il mount automatico:

    text
    UUID=58C7-89DF /mnt/usb exfat defaults,uid=1000,gid=1000 0 0

  3. Monta la chiavetta:

    bash
    sudo mount -a

  4. Controlla che sia visibile:

    bash
    ls /mnt/usb

6. Verifica del backup

  1. Apri Duplicati via browser (http://<IP_RPi>:8200).

  2. Assicurati che i backup sulla chiavetta siano visibili.

  3. Controlla che la password e la chiave di crittografia siano corrette.

7. Ripristino dei dati

  1. Dal pannello Duplicati, crea un job di ripristino.

  2. Seleziona come sorgente il backup sulla chiavetta.

  3. Imposta come destinazione il percorso originale sul nuovo sistema (/DATA, /home/ulisse, ecc.).

  4. Avvia il ripristino.
    ⚠️ Nota: Se file come database di Portainer o Home Assistant sono in uso, il ripristino potrebbe fallire. In quel caso, ferma i container prima:

    bash
    sudo docker stop <container>

  5. Controlla i log di Duplicati per eventuali errori.

8. Avvio dei servizi

  1. Una volta ripristinati tutti i dati, riavvia CasaOS e i container:

    bash
    sudo systemctl restart casaos
sudo docker start <container>

  2. Verifica che Home Assistant e gli altri servizi siano accessibili e funzionanti.

9. Test del ripristino

  1. Accedi a Home Assistant e controlla che i tuoi dati e configurazioni siano intatti.

  2. Controlla eventuali container Docker rimappati (come Duplicati) e la corretta visibilità dei dati.

10. Suggerimenti

  • Mantieni la chiavetta collegata periodicamente per backup automatici.

  • Non usare Duplicati come root, meglio usare un utente con permessi controllati e concedere accesso alle cartelle.

  • Esegui backup incrementali frequenti (es. 12h) e backup completo settimanale.

Se vuoi, posso prepararti una versione “rapida da seguire” con comandi precisi da terminale, pronta da copiare/incollare, così non devi interpretare ogni passaggio.

Perfetto! Ti preparo una mappa visiva della configurazione di Duplicati, con i volumi, mount point, sorgenti e destinazioni dei backup, così da avere tutto chiaro a colpo d’occhio.

Schema visivo Duplicati su CasaOS

bash
┌──────────────────────────┐
│       Raspberry Pi       │
│   (Debian 12 / CasaOS)  │
└─────────────┬────────────┘
              │
              │ Docker (Host Network)
              ▼
      ┌───────────────────┐
      │   Container       │
      │   Duplicati       │
      │   port 8200       │
      └─────────┬─────────┘
                │
                │
                │ Source Volumes (inside container)
                │
┌───────────────┴───────────────┐
│ /source                         │
│ ├─ /DATA                        │  ← /DATA dal RPi, include AppData, Home Assistant, ecc.
│ │   ├─ AppData/duplicati/config │  ← Config Duplicati
│ │   └─ AppData/homeassistant    │  ← Database HA, ecc.
│ ├─ /home                        │  ← Home dell’utente
│ └─ /etc                         │  ← Config di sistema
└─────────────────────────────────┘

                │
                │ Backup Destination
                │
┌───────────────┴───────────────┐
│ /backups                       │
│ └─ /mnt/usb/RPiBackup          │  ← Chiavetta USB o cartella di backup
└─────────────────────────────────┘

Legenda

  • /source → tutte le directory del Raspberry da includere nel backup.

  • /backups → destinazione dei backup, fisica o USB.

  • DUPLICATI gira nel container, accessibile su porta 8200.

  • Frequenza backup: configurata in Duplicati (es. ogni 12h).
